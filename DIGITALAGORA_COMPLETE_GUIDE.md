# ЦИФРОВАЯ АГОРА – полное руководство по проекту (всё в одном файле)

## 1. О проекте

**DigitalAgora** – децентрализованная платформа для законотворческой 
деятельности акционеров (жителей).  
Позволяет:
- регистрироваться акционерам (фамилия, имя, отчество, дата рождения, 
место жительства, кошелёк, email, пароль);
- входить по email/паролю (JWT + bcrypt);
- создавать и обсуждать предложения по улучшению;
- голосовать токенами через смарт-контракт (Ethereum);
- анализировать идеи с помощью ИИ (OpenAI, заглушка);
- находить подрядчиков (демо-список);
- контролировать финансирование проектов.

## 3. Архитектура (как всё работает)

### 3.1. Общая схема взаимодействия
[Пользователь] → [Браузер] → [Статический сервер (http-server)]
↓
[index.html + JS]
↓
┌───────────────┼───────────────┐
↓ ↓ ↓
[Бэкенд API] [MetaMask] [Блокчейн-нода]
(порт 5000) (кошелёк) (порт 8545)


### 3.2. Пошаговое описание работы

**Шаг 1. Загрузка страницы**  
- Открывается `http://localhost:3000` → статический сервер отдаёт `index.html`.  
- JS проверяет `localStorage`: если есть `token` и `agora_user` → сразу дашборд; 
иначе → форма регистрации/входа.

**Шаг 2. Регистрация**  
- Заполняются 8 полей → POST на `http://localhost:5000/api/auth/register`.  
- Бэкенд проверяет email, хеширует пароль (bcrypt), сохраняет в `data/users.json`, 
возвращает JWT.  
- Фронтенд сохраняет токен и данные пользователя → показывает дашборд.

**Шаг 3. Вход** – аналогично через `/api/auth/login`.

**Шаг 4. Подключение MetaMask**  
- Кнопка «Подключить кошелёк» → `ethereum.request({ method: 'eth_requestAccounts' })`.  
- После разрешения создаётся `Web3Provider`, подписываются контракты.  
- Отображается адрес кошелька и баланс токенов.

**Шаг 5. Создание предложения (блокчейн)**  
- Пользователь вводит название и описание → вызов `governor.propose(...)`.  
- Транзакция через MetaMask подтверждается, майнится.  
- Событие `ProposalCreated` даёт ID предложения; оно сохраняется в локальном кэше.

**Шаг 6. Голосование**  
- Нажатие «За» или «Против» → `governor.castVote(proposalId, support)`.  
- Транзакция отправляется, голос засчитывается (вес = количество токенов).

**Шаг 7. Чтение предложений из блокчейна (планируется)**  
- В будущем вместо `localStorage` будут использоваться события `ProposalCreated`.

## 4. Инструкция по запуску (для разработчика)

### 4.1. Установка зависимостей

```bash
# Корень проекта (Hardhat)
npm install

# Бэкенд
cd backend
npm install express jsonwebtoken bcrypt dotenv cors
cd ..

# Фронтенд (статический, ничего устанавливать не надо)


4.2. Переменные окружения
Создайте backend/.env:


PORT=5000
JWT_SECRET=суперсекретный_ключ_12345

4.3. Запуск компонентов (4 терминала)
Терминал 1 – блокчейн-нода

bash
npx hardhat node

Терминал 2 – деплой контрактов

bash
npx hardhat run scripts/deploy.js --network localhost
Скопируйте адреса из deployed.json.

Терминал 3 – бэкенд

bash
cd backend && node server.js
Терминал 4 – фронтенд

bash
cd frontend/public && npx http-server -p 3000

4.4. Работа в браузере
Открыть http://localhost:3000

Зарегистрироваться (все поля обязательны, кроме отчества)

Войти

В дашборде нажать «Подключить кошелёк»

В MetaMask добавить сеть http://localhost:8545 (Chain ID 31337)

Импортировать один из тестовых аккаунтов (ключи видны при запуске npx hardhat node)
